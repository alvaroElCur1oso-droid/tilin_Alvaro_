  1. ¿Qué es PSeInt?

    2. PSeInt es un intérprete de pseudocódigo que te permite escribir algoritmos en un lenguaje similar al español. Su nombre viene de Pseudocódigo Se Intenta. Es ideal para estudiantes que están aprendiendo los fundamentos de la programación, como variables, estructuras de control y operadores.

  2. Los Cimientos: Estructura, Variables y Tipos de Datos

    3. Antes de escribir cualquier código, es fundamental entender la estructura y cómo se guardan los datos.

    La Estructura de un Algoritmo

    Todo programa en PSeInt empieza y termina de la misma manera. Es tu lienzo en blanco.

    
Proceso NombreDelAlgoritmo
    // Aquí es donde sucede la magia:
    // Puedes definir variables, pedir datos,
    // hacer cálculos y mostrar resultados.
FinProceso

    Variables y Tipos de Datos

    Las variables son como cajas en la memoria de la computadora donde puedes guardar información. Cada caja solo puede guardar un tipo específico de cosa. Para crear una, usas el comando Definir.

    • Entero: Números sin decimales (1, 100, -5).
    • Real: Números con decimales (3.14, 0.5, -99.9).
    • Caracter: Un solo carácter o una cadena de texto ( "Hola", "a", "Este es un ejemplo" ).
    • Logico: Solo puede tener dos valores: Verdadero o Falso. Es esencial para tomar decisiones.
    Proceso NombreDelAlgoritmo
    // Asignamos valores a las variables
    edad <- 25;
    precio <- 19.99;
    nombre <- "Ana";

    // Un ejemplo con el tipo Lógico
    esMayorDeEdad <- (edad >= 18); // Esto asignará Verdadero

    Escribir "Nombre: ", nombre;
    Escribir "Edad: ", edad;
    Escribir "Es mayor de edad: ", esMayorDeEdad;
FinProceso
  3. Interactuando con el Usuario: Entrada y Salida

    4. Un programa es útil cuando puede comunicarse. Escribir y Leer son tus comandos para hacerlo.

    • Escribir: Muestra un mensaje en pantalla. Puedes combinar texto (entre comillas) con el valor de las variables usando comas.
    • Escribir Sin Saltar: Similar a Escribir, pero no agrega un salto de línea al final, lo que es útil para crear tablas o menús.
    • Leer: Pide al usuario que ingrese un valor y lo guarda en una variable.
    Proceso SaludoPersonalizado
    Definir nombre Como Caracter;
    Escribir "Por favor, escribe tu nombre:";
    Leer nombre;
    Escribir "¡Hola, ", nombre, "! Bienvenido a PSeInt.";
FinProceso
  4. Lógica y Operadores: El Corazón del Programa

    5. Los operadores te permiten hacer cálculos y comparar valores.

    • Aritméticos: Para cálculos matemáticos.
      • +, -, *, /: Suma, resta, multiplicación, división.
      • ^: Potencia.
      • %: Módulo (devuelve el residuo de una división).
    • Relacionales: Para comparar valores. El resultado siempre es Verdadero o Falso.
      • >, <: Mayor que, menor que.
      • >=, <=: Mayor o igual, menor o igual.
      • ==: Igual a.
      • <>: Diferente de.
    • Lógicos: Para combinar condiciones.
      • Y: condicion1 Y condicion2 es Verdadero solo si ambas son Verdadero.
      • O: condicion1 O condicion2 es Verdadero si al menos una es Verdadero.
      • No: Invierte el valor lógico (No Verdadero es Falso).
  5. Controlando el Flujo: Condicionales y Bucles

    6. Estos comandos son lo que le da "inteligencia" a un programa, permitiéndole tomar decisiones y repetir tareas.

    Decidiendo con Si, SinoSi y Según

    • Decidiendo con Si, SinoSi y Según
    • SinoSi: Una extensión del Si. Te permite evaluar múltiples condiciones de forma ordenada.
    • Según: Ideal cuando tienes una variable que puede tener muchos valores fijos.
    Proceso EjemploDecision
    Definir nota Como Entero;
    Escribir "Ingresa tu nota (1-10):";
    Leer nota;

    Si nota >= 9 Entonces
        Escribir "Excelente, tu nota es sobresaliente.";
    SinoSi nota >= 7 Entonces
        Escribir "Muy bien, tu nota es buena.";
    Sino
        Escribir "Necesitas mejorar.";
    FinSi
FinProceso

    Repitiendo Tareas con Para, Mientras y Repetir

    • Para: Úsalo cuando sabes exactamente cuántas veces quieres repetir algo. Es perfecto para recorrer listas de datos.
    • Mientras: Repite un bloque de código mientras la condición sea Verdadero. La condición se verifica al inicio, así que si es falsa desde el principio, el código nunca se ejecuta.
    • Repetir-Hasta Que: Similar a Mientras, pero la condición se verifica al final. Esto garantiza que el código se ejecute al menos una vez.
  6. Estructuras de Datos Avanzadas: Arreglos

    7. Un arreglo es una variable que puede almacenar una lista de valores del mismo tipo. Es como una estantería con varias casillas numeradas.

    • Vectores: Son arreglos de una sola dimensión (una lista). Se declaran con Dimension.
    • Matrices: Son arreglos de dos dimensiones (una tabla).
    Proceso EjemploVectores
    // Creamos un vector para 5 notas
    Dimension notas[5];
    Definir i Como Entero;

    // Llenamos el vector pidiéndole al usuario las notas
    Para i <- 1 Hasta 5 Con Paso 1 Hacer
        Escribir "Ingresa la nota ", i, ":";
        Leer notas[i];
    FinPara

    // Calculamos el promedio
    Definir suma Como Real;
    suma <- 0;
    Para i <- 1 Hasta 5 Con Paso 1 Hacer
        suma <- suma + notas[i];
    FinPara

    Escribir "El promedio de las notas es: ", suma / 5;
FinProceso
  7. Modularizando el Código: Subprocesos

    8. Cuando tu programa crece, se vuelve difícil de manejar. Los subprocesos te permiten dividir el código en tareas más pequeñas y reutilizables, como si fueran mini-programas.

    • Procedimientos: Realizan una tarea pero no devuelven un valor. Son perfectos para mostrar un mensaje o un menú.
    • Funciones: Realizan una tarea y devuelven un valor. Son ideales para cálculos.
    // Un Subproceso (Función) que calcula el área de un círculo
SubProceso resultado <- CalcularArea(radio)
    Definir PI Como Real;
    PI <- 3.14159;
    resultado <- PI * radio * radio;
FinSubProceso

// El proceso principal que usa la función
Proceso UsarFuncion
    Definir radio_circulo, area Como Real;
    Escribir "Ingresa el radio del círculo:";
    Leer radio_circulo;

    // Llamamos a la función y guardamos el resultado
    area <- CalcularArea(radio_circulo);

    Escribir "El área del círculo es: ", area;
FinProceso
  8. Herramientas de Visualización y Depuración

    9. PSeInt tiene dos características que lo hacen inigualable para aprender:

    • Diagrama de Flujo: Haz clic en el ícono del diagrama de flujo en la barra de herramientas. Esto generará una representación visual de tu programa. Verás bloques, flechas y decisiones. Es la forma más clara de entender la lógica de un algoritmo.
    • Ejecutar Paso a Paso: Haz clic en el ícono de las huellas de un pie. Esto ejecutará tu código línea por línea, mostrándote exactamente qué hace el programa y cómo cambian los valores de las variables. Es una herramienta poderosa para encontrar errores.
